# Project Claw 后端故障排查指南

## 🔧 常见问题解决

### 问题 1: ModuleNotFoundError: No module named 'auth_guard'

**原因**: 缺少 `auth_guard.py` 模块

**解决方案**:
```bash
# 确保 auth_guard.py 在 cloud_server 目录中
ls cloud_server/auth_guard.py

# 如果不存在，创建该文件
# 文件已在 cloud_server/auth_guard.py 中提供
```

### 问题 2: ModuleNotFoundError: No module named 'model_gateway'

**原因**: 缺少 `model_gateway.py` 模块

**解决方案**:
```bash
# 确保 model_gateway.py 在 cloud_server 目录中
ls cloud_server/model_gateway.py

# 如果不存在，创建该文件
# 文件已在 cloud_server/model_gateway.py 中提供
```

### 问题 3: ModuleNotFoundError: No module named 'cost_tracker'

**原因**: 缺少 `cost_tracker.py` 模块

**解决方案**:
```bash
# 确保 cost_tracker.py 在 cloud_server 目录中
ls cloud_server/cost_tracker.py

# 如果不存在，创建该文件
# 文件已在 cloud_server/cost_tracker.py 中提供
```

### 问题 4: pip install 失败

**原因**: 网络问题或依赖冲突

**解决方案**:
```bash
# 使用清华大学镜像源
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt

# 或使用阿里云镜像源
pip install -i https://mirrors.aliyun.com/pypi/simple/ -r requirements.txt

# 升级 pip
python -m pip install --upgrade pip
```

### 问题 5: Port 8765 already in use

**原因**: 端口已被占用

**解决方案**:
```bash
# 查找占用端口的进程 (Windows)
netstat -ano | findstr :8765

# 杀死进程
taskkill /PID <PID> /F

# 或使用不同的端口
python -m uvicorn api_server_pro:app --reload --port 8766
```

### 问题 6: ImportError: cannot import name 'initialize_model_gateway'

**原因**: 模块导入路径错误

**解决方案**:
```bash
# 确保在 cloud_server 目录中运行
cd cloud_server

# 检查 __init__.py 是否存在
touch __init__.py

# 重新启动服务
python -m uvicorn api_server_pro:app --reload --port 8765
```

## 📋 完整的启动步骤

### 第 1 步: 检查文件结构

```bash
cd d:\桌面\Project Claw\cloud_server

# 确保以下文件存在
ls api_server_pro.py
ls auth_guard.py
ls model_gateway.py
ls cost_tracker.py
ls requirements.txt
```

### 第 2 步: 创建虚拟环境

```bash
# Windows
python -m venv venv
venv\Scripts\activate.bat

# Linux/Mac
python -m venv venv
source venv/bin/activate
```

### 第 3 步: 安装依赖

```bash
pip install -r requirements.txt
```

### 第 4 步: 启动服务

```bash
# 方式 1: 使用启动脚本 (Windows)
start.bat

# 方式 2: 直接运行
python -m uvicorn api_server_pro:app --reload --port 8765

# 方式 3: 使用 Python 直接运行
python api_server_pro.py
```

### 第 5 步: 验证服务

```bash
# 打开浏览器访问
http://localhost:8765/health

# 或使用 curl
curl http://localhost:8765/health
```

## 🔍 调试技巧

### 启用详细日志

```python
# 在 api_server_pro.py 中修改日志级别
import logging

logging.basicConfig(
    level=logging.DEBUG,  # 改为 DEBUG
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

### 使用 Python 调试器

```bash
# 安装 pdb
pip install pdbpp

# 在代码中添加断点
import pdb; pdb.set_trace()

# 运行服务
python api_server_pro.py
```

### 检查依赖版本

```bash
# 列出所有已安装的包
pip list

# 检查特定包的版本
pip show fastapi
```
