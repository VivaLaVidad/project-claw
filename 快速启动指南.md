# Project Claw 完整系统启动指南

## 🚀 一键启动 (推荐)

### Windows 用户

```bash
# 打开 PowerShell 或 CMD，进入项目目录
cd d:\桌面\Project\ Claw\cloud_server

# 运行启动脚本
start.bat
```

### Linux/Mac 用户

```bash
# 进入项目目录
cd ~/Desktop/Project\ Claw/cloud_server

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python -m uvicorn api_server_pro:app --reload --port 8765
```


**验证虚拟环境已激活**:

```
INFO:     Started reloader process [12345] using WatchFiles
```

## 🛠️ 常见问题快速解决

### 问题: Port 8765 already in use

```bash
# 查找占用端口的进程
netstat -ano | findstr :8765

# 杀死进程 (替换 PID)
taskkill /PID 12345 /F

# 或使用不同的端口
python -m uvicorn api_server_pro:app --reload --port 8766
```

### 问题: ModuleNotFoundError

```bash
# 确保在 cloud_server 目录中
cd d:\桌面\Project\ Claw\cloud_server

# 检查虚拟环境是否激活
# 应该看到 (venv) 前缀

# 重新安装依赖
pip install -r requirements.txt
```

### 问题: npm install 失败

```bash
# 清除 npm 缓存
npm cache clean --force

# 使用淘宝镜像
npm install -g cnpm --registry=https://registry.npm.taobao.org
cnpm install

# 或直接重试
npm install
```

## 🚨 故障排查

如果遇到问题，按以下步骤排查:

1. **检查文件是否存在**
   ```bash
   ls cloud_server/auth_guard.py
   ls cloud_server/model_gateway.py
   ls cloud_server/cost_tracker.py
   ```

2. **检查虚拟环境**
   ```bash
   # 应该看到 (venv) 前缀
   which python
   ```

3. **检查依赖**
   ```bash
   pip list | grep fastapi
   ```

4. **查看详细日志**
   ```bash
   python -m uvicorn api_server_pro:app --reload --port 8765 --log-level debug
   ```

5. **查看完整的故障排查指南**
   ```bash
   cat 后端故障排查指南.md
   ```
