# Project Claw v9.0 - 完整项目介绍

## 一、项目概述

**Project Claw** 是一个工业级的自动回复系统，专门为微信/聊天应用设计。通过 OCR 识别、AI 回复和飞书同步，实现完全自动化的消息处理流程。

### 核心目标
- 自动识别用户消息
- 智能生成回复内容
- 实时同步到飞书
- 保证消息准确性和稳定性

## 三、核心模块详解

### 3.1 消息识别模块（MessageExtractor）

#### 工作流程

```
原始 OCR 结果
    ↓
文本清洗（去换行、多余空格）
    ↓
噪声过滤（时间、符号、UI 文案）
    ↓
气泡检测（判断是否用户消息）
    ↓
位置追踪（检测是否新消息）
    ↓
返回最新用户消息
```

#### 关键算法

**1. 文本清洗**
```python
def clean_text(text):
    # 去掉换行符
    text = text.replace("\n", " ").replace("\r", " ")
    # 去掉多余空格
    text = re.sub(r"\s+", " ", text)
    return text.strip()
```

**2. 噪声过滤**
```python
def is_noise_text(text):
    # 过滤 UI 关键词
    if text in {"发送", "表情", "语音", ...}:
        return True
    # 过滤时间（14.51、14:51）
    if re.fullmatch(r"\d{1,2}[:：.]\d{2}", text):
        return True
    # 过滤单字符符号
    if len(text) == 1 and not re.search(r"[\u4e00-\u9fffA-Za-z0-9]", text):
        return True
    return False
```

**3. 位置追踪**
```python
def is_new_message(text, bbox):
    # 内容不同 → 新消息
    if text != self.last_message_text:
        return True
    
    # 位置变化超过 50 像素 → 新消息
    distance = abs(bbox_bottom(bbox) - bbox_bottom(self.last_message_bbox))
    return distance > 50
```

#### 为什么需要位置追踪？

**问题**：同一条消息在屏幕上停留多轮，OCR 每次都能识别出来
**解决**：记录消息的 bbox 位置，只有位置变化才认为是新消息
**效果**：完全消除"自动重复回复"问题

### 3.2 气泡检测模块（BubbleDetector）

#### 原理

微信消息气泡有不同颜色：
- **用户消息**：灰色气泡（左边）
- **机器人回复**：绿色气泡（右边）

通过 HSV 色彩空间检测，区分两种消息。

#### 实现

```python
def is_user_message(bbox, img):
    # 提取气泡区域
    bubble_region = img[y_min:y_max, x_min:x_max]
    
    # 转换到 HSV 色彩空间
    hsv = cv2.cvtColor(bubble_region, cv2.COLOR_BGR2HSV)
    
    # 检测用户气泡（灰色）
    user_mask = cv2.inRange(hsv, user_lower, user_upper)
    user_ratio = np.count_nonzero(user_mask) / user_mask.size
    
    # 检测机器人气泡（绿色）
    bot_mask = cv2.inRange(hsv, bot_lower, bot_upper)
    bot_ratio = np.count_nonzero(bot_mask) / bot_mask.size
    
    # 绿色占比 > 20% → 机器人消息 → 过滤
    return bot_ratio <= 0.2
```

#### 为什么需要气泡检测？

**问题**：OCR 会识别屏幕上所有文本，包括机器人自己的回复
**解决**：通过气泡颜色区分，只处理用户消息
**效果**：避免机器人回复自己的消息

### 3.3 消息去重模块（MessageDedup）

#### 工作原理

```
消息文本
    ↓
计算 MD5 hash
    ↓
检查缓存中是否存在
    ↓
检查时间是否在窗口内
    ↓
返回是否重复
```

#### 实现

```python
def is_duplicate(text):
    msg_hash = hashlib.md5(text.encode("utf-8")).hexdigest()
    now = time.time()
    
    for item in self.cache:
        # 内容相同 AND 时间在窗口内 → 重复
        if (now - item["time"] < self.time_window and 
            item["hash"] == msg_hash):
            return True
    
    return False
```

#### 为什么需要去重？

**问题**：即使位置追踪，仍可能因为网络延迟等原因重复识别
**解决**：基于内容 hash + 时间窗口的双重去重
**效果**：120 秒内相同消息只回复一次

### 3.4 LLM 回复模块（ask_llm）

#### 工作流程

```
用户消息
    ↓
构建 API 请求
    ↓
调用 DeepSeek API
    ↓
解析回复内容
    ↓
返回回复文本
```

#### 实现

```python
def ask_llm(msg):
    url = "https://api.deepseek.com/chat/completions"
    
    data = {
        "model": "deepseek-chat",
        "messages": [
            {"role": "system", "content": "你是一个热情的店老板..."},
            {"role": "user", "content": msg}
        ],
        "temperature": 0.7,
        "max_tokens": 80
    }
    
    res = requests.post(url, json=data, timeout=10)
    return res.json()["choices"][0]["message"]["content"].strip()
```

#### 为什么用 DeepSeek？

- **成本低**：API 价格便宜
- **速度快**：响应时间短
- **效果好**：中文理解能力强
- **稳定性**：服务可用性高

## 四、核心思路

### 4.1 消息识别的三层防护

```
第一层：OCR 识别
    ↓ 识别所有屏幕文本
    
第二层：气泡检测
    ↓ 过滤机器人自己的消息
    
第三层：位置追踪 + 去重
    ↓ 防止重复回复
    
最终：准确的用户消息
```

### 4.2 性能优化的三个关键

**1. 局部 OCR**
- 不识别整个屏幕
- 只识别聊天区域（15% ~ 82% 高度）
- 减少 OCR 计算量 60%+

**2. 异步同步**
- 消息发送后立即返回
- 飞书同步在后台进行
- 不阻塞主循环

**3. 智能轮询**
- 无新消息：0.5 秒轮询
- 有新消息：立即处理
- 异常情况：1 秒重试

### 4.3 稳定性的三个保障

**1. 异常恢复**
```
OCR 失败 → 重试 → 连续失败 3 次 → 重新初始化
```

**2. 详细日志**
```
每个关键步骤都有日志记录
便于问题排查和性能分析
```

**3. 配置灵活性**
```
所有参数都可配置
无需修改代码即可调整行为
```
