### 1. 小程序环境检测（miniprogram/app.js）

```javascript
_getServerBase() {
  const envVersion = wx.getAccountInfoSync().miniProgram.envVersion;
  const isDev = envVersion === 'develop';
  
  if (isDev) {
    // 本地开发：使用 localhost
    console.log('[App] 环境: 开发环境 (localhost)');
    return 'http://localhost:8765';
  } else {
    // 生产环境：使用 Railway 部署的服务器
    console.log('[App] 环境: 生产环境 (Railway)');
    return 'https://project-claw-production.up.railway.app';
  }
}

_getWSBase() {
  const envVersion = wx.getAccountInfoSync().miniProgram.envVersion;
  const isDev = envVersion === 'develop';
  
  if (isDev) {
    console.log('[App] WebSocket: 开发环境 (localhost)');
    return 'ws://localhost:8765';
  } else {
    console.log('[App] WebSocket: 生产环境 (Railway)');
    return 'wss://project-claw-production.up.railway.app';
  }
}
```

## 🚨 常见问题

### Q1：生产环境显示"服务器离线"
```
解决：
1. 检查 Railway 部署状态
2. 访问 https://project-claw-production.up.railway.app/health
3. 检查后端日志
4. 重新部署
```

### Q2：小程序无法连接到生产环境
```
解决：
1. 检查网络连接
2. 检查 API 地址是否正确
3. 检查防火墙设置
4. 检查 CORS 配置
```

### Q3：生产环境 API 返回 404
```
解决：
1. 检查后端路由是否正确
2. 检查 API 端点是否存在
3. 查看后端日志
4. 重新部署
```

### Q4：WebSocket 连接失败
```
解决：
1. 检查 WebSocket 地址是否正确
2. 检查防火墙是否允许 WebSocket
3. 检查后端 WebSocket 配置
4. 重新部署
```
